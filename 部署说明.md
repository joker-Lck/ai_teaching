## 🖥️ 系统要求

### 硬件要求

| 组件 | 最低配置 | 推荐配置 | 说明 |
|------|----------|----------|------|
| **CPU** | 双核 2.0 GHz | 四核 3.0 GHz+ | 无需 GPU |
| **内存** | 4 GB | 8 GB+ | 运行 Python + MySQL |
| **硬盘** | 10 GB 可用空间 | 50 GB+ | 数据库 + 日志 + 缓存 |
| **网络** | 稳定互联网连接 | 10 Mbps+ | 访问 Kimi API |

### 软件要求

| 软件 | 版本要求 | 下载地址 | 说明 |
|------|----------|----------|------|
| **Python** | 3.8 ~ 3.11 | https://www.python.org/ | ⚠️ 不支持 3.12+ |
| **MySQL** | 8.0+ | https://dev.mysql.com/downloads/ | 需要 JSON 字段支持 |
| **pip** | 20.0+ | Python 自带 | Python 包管理器 |

### 支持的操作系统

- ✅ **Windows**: Windows 10/11（64 位）
- ✅ **macOS**: macOS 12+ (Monterey)
- ✅ **Linux**: Ubuntu 20.04+ / CentOS 8+

### 版本说明

**Python 版本**：
- ✅ **推荐版本**: Python 3.10.6（经过完整测试）
- ✅ **兼容版本**: Python 3.8 ~ 3.11
- ❌ **不支持**: Python 3.12+（部分依赖库不兼容）
- ❌ **不支持**: Python 3.7-（已停止维护）

**MySQL 版本**：
- ✅ **推荐版本**: MySQL 8.0.33+（支持 JSON 字段优化）
- ✅ **兼容版本**: MySQL 5.7+（部分功能受限）
- ❌ **不支持**: MySQL 5.6-（不支持 JSON 字段）

**检查版本**：
```bash
python --version    # 应显示: Python 3.8.x ~ 3.11.x
mysql --version     # 应显示: mysql Ver 8.0.x 或 mysql Ver 5.7.x
```

### 步骤 2: 配置环境变量

#### 🔐 安全提示

**重要**：为了保护您的 API Key 安全，请遵循以下最佳实践：

1. ✅ **不要硬编码 API Key**：必须通过 `.env` 文件配置
2. ✅ **`.env` 文件已被 Git 忽略**：不会上传到版本控制系统
3. ✅ **定期轮换 API Key**：建议每 3-6 个月更换一次
4. 🛡️ **设置使用限额**：在 Kimi 平台限制每日/每月用量
5. 🔒 **生产环境**：建议使用密钥管理服务（如 AWS Secrets Manager）

#### 配置方法

1. **复制环境变量模板**：
```bash
# Windows
copy .env.example .env

# macOS/Linux
cp .env.example .env
```

2. **编辑 `.env` 文件**：
```bash
# Kimi AI 配置
KIMI_API_KEY=sk-your_actual_api_key_here
KIMI_BASE_URL=https://api.moonshot.cn/v1

# 主业务数据库
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_actual_password
DB_NAME=ai_teaching_assistant

# 答疑数据库
QA_DB_HOST=localhost
QA_DB_PORT=3306
QA_DB_USER=root
QA_DB_PASSWORD=your_actual_password
QA_DB_NAME=ai_qa_records

# RAG 知识库数据库
RAG_DB_HOST=localhost
RAG_DB_PORT=3306
RAG_DB_USER=root
RAG_DB_PASSWORD=your_actual_password
RAG_DB_NAME=ai_rag_knowledge
```

**⚠️ 注意事项**：
- 将 `your_actual_api_key_here` 替换为您的真实 Kimi API Key
- 将 `your_actual_password` 替换为您的 MySQL 密码
- `.env` 文件包含敏感信息，**切勿上传到公开代码仓库**

## 💾 数据库初始化

### 创建数据库

```bash
# 登录 MySQL
mysql -u root -p

# 创建三个数据库
CREATE DATABASE IF NOT EXISTS ai_teaching_assistant CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS ai_qa_records CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS ai_rag_knowledge CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 退出 MySQL
exit;
```

### 运行初始化脚本

```bash
# 1. 初始化主业务数据库（users, classes, students, courseware, learning_analysis）
python init_db.py

# 2. 初始化答疑专用数据库（qa_records, qa_sessions, interaction_records）
python init_qa_db.py

# 3. 初始化 RAG 知识库数据库（knowledge_documents，预置 93 本电子书）
python init_rag_db.py
```

**💡 提示**：
- RAG 数据库已预置 93 本电子书（AI, C++, Extra, Math, Python 五大类）
- 系统采用 RAG 优先策略：提问时先查知识库，减少 AI 调用
- 数据库字符集统一使用 `utf8mb4`（支持 emoji 和生僻字）

### 验证数据库初始化

```bash
# 检查表结构
mysql -u root -p ai_teaching_assistant -e "SHOW TABLES;"
mysql -u root -p ai_qa_records -e "SHOW TABLES;"
mysql -u root -p ai_rag_knowledge -e "SHOW TABLES;"
```

**预期输出**：
- `ai_teaching_assistant`: users, classes, students, courseware, learning_analysis
- `ai_qa_records`: qa_records, qa_sessions, interaction_records
- `ai_rag_knowledge`: knowledge_documents

## 🔍 故障排查

### 问题 1: 依赖安装失败

**症状**：
```
ERROR: Could not find a version that satisfies the requirement xxx
```

**解决方案**：
```bash
# 升级 pip
python -m pip install --upgrade pip

# 清除缓存重装
pip cache purge
pip install -r requirements.txt --force-reinstall

# 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

---

### 问题 2: MySQL 连接失败

**症状**：
```
mysql.connector.errors.DatabaseError: 2003 (HY000): Can't connect to MySQL server
```

**检查清单**：
1. MySQL 服务是否启动？
   ```bash
   # Windows
   net start | findstr MySQL
   
   # macOS
   brew services list | grep mysql
   
   # Linux
   systemctl status mysql
   ```

2. 用户名密码是否正确？
   ```bash
   mysql -u root -p -h localhost -P 3306
   ```

3. 防火墙是否阻止 3306 端口？
   ```bash
   # Windows: 检查防火墙设置
   # Linux: sudo ufw allow 3306
   ```

4. 数据库是否已创建？
   ```bash
   mysql -u root -p -e "SHOW DATABASES;"
   ```

---

### 问题 3: API Key 无效（401 错误）

**症状**：
```
openai.AuthenticationError: Error code: 401 - Invalid API key
```

**解决方案**：
1. 确认 `.env` 文件存在且配置正确
2. 检查 API Key 是否复制完整（无空格、换行）
3. 确认 API Key 未过期
4. 查看 Kimi 账户余额是否充足
5. 重启应用使环境变量生效

---

### 问题 4: Python 版本不兼容

**症状**：
```
SyntaxError: invalid syntax
ImportError: cannot import name 'xxx' from 'yyy'
```

**解决方案**：
```bash
# 检查当前 Python 版本
python --version

# 如果版本 > 3.11，请安装 Python 3.10 或 3.11
# 然后重新安装依赖
```

## 🏭 生产环境部署

### 1. 服务器要求

**推荐配置**：
- CPU: 4 核 3.0 GHz+
- 内存: 8 GB+
- 硬盘: 100 GB SSD
- 网络: 100 Mbps+
- 操作系统: Ubuntu 20.04 LTS / CentOS 8

### 2. 使用 Docker 部署（推荐）

**创建 `Dockerfile`**：
```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

COPY . .

EXPOSE 8501

CMD ["streamlit", "run", "streamlit_001.py", "--server.address", "0.0.0.0"]
```

**创建 `docker-compose.yml`**：
```yaml
version: '3'

services:
  mysql:
    image: mysql:8.0
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"

  app:
    build: .
    restart: always
    ports:
      - "8501:8501"
    environment:
      - KIMI_API_KEY=${KIMI_API_KEY}
      - DB_HOST=mysql
      - DB_PASSWORD=${DB_PASSWORD}
      - QA_DB_HOST=mysql
      - QA_DB_PASSWORD=${DB_PASSWORD}
      - RAG_DB_HOST=mysql
      - RAG_DB_PASSWORD=${DB_PASSWORD}
    depends_on:
      - mysql

volumes:
  mysql_data:
```

**启动服务**：
```bash
docker-compose up -d
```

### 3. 使用 Nginx 反向代理

**Nginx 配置**：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8501;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 4. 启用 HTTPS

```bash
# 使用 Let's Encrypt 免费证书
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 5. 数据库备份策略

**自动备份脚本**：
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份三个数据库
mysqldump -u root -p${DB_PASSWORD} ai_teaching_assistant > $BACKUP_DIR/teaching_$DATE.sql
mysqldump -u root -p${DB_PASSWORD} ai_qa_records > $BACKUP_DIR/qa_$DATE.sql
mysqldump -u root -p${DB_PASSWORD} ai_rag_knowledge > $BACKUP_DIR/rag_$DATE.sql

# 压缩备份文件
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz $BACKUP_DIR/*.sql

# 删除 30 天前的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +30 -delete

echo "Backup completed: $DATE"
```

**设置定时任务**：
```bash
# 每天凌晨 2 点执行备份
crontab -e
0 2 * * * /path/to/backup.sh >> /var/log/mysql_backup.log 2>&1
```
