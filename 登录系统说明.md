# 登录系统使用说明

## 📋 功能概述

多模态 AI 教学智能体现在支持用户登录系统，所有用户账号信息存储在 MySQL 数据库中。

### 核心功能

- ✅ **用户注册**：支持教师和学生两种角色
- ✅ **用户登录**：用户名 + 密码登录
- ✅ **游客模式**：无需注册即可体验基本功能
- ✅ **密码加密**：使用 SHA-256 + Salt 加密存储
- ✅ **角色管理**：教师、学生、管理员三种角色
- ✅ **退出登录**：一键退出，返回登录界面

## 🚀 快速开始

### 1. 升级数据库（仅首次需要）

如果使用的是现有数据库，需要执行升级脚本添加密码字段：

**登录页面功能**：
- **登录**：输入用户名和密码
- **注册**：新用户注册账号
- **游客模式**：无需账号直接体验

## 🔐 密码安全

### 加密机制

- **算法**：SHA-256
- **Salt**：16 字节随机盐值
- **存储格式**：`salt$hash`
- **特点**：
  - 相同密码每次加密结果不同（因 salt 不同）
  - 防止彩虹表攻击
  - 防止密码泄露后被破解

### 密码要求

- 长度：至少 6 位
- 建议使用字母 + 数字组合
- 避免使用常见密码（如 123456、password 等）

## 📝 使用场景

### 场景 1：新用户注册

1. 打开登录页面
2. 点击" 注册"标签
3. 填写信息：
   - 用户名（3-20 个字符）
   - 密码（至少 6 位）
   - 确认密码
   - 邮箱（可选）
   - 角色（教师/学生）
4. 点击"注册"按钮
5. 注册成功后返回登录

### 场景 2：用户登录

1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后进入主应用

### 场景 3：游客模式

1. 点击“游客模式”按钮
2. 直接进入主应用
3. **功能限制**：
   - ✅ **可用功能**：智能答疑（仅可提问和查看AI回复）
   - ❌ **不可用功能**：
     - 课件生成
     - 学情分析
     - 知识库管理
   - ⚠️ **限制说明**：
     - 无法保存聊天记录
     - 无法导出学习数据
     - 无法使用个性化功能
     - 侧边栏会显示游客模式提示

### 场景 4：退出登录

1. 在侧边栏顶部看到用户信息
2. 点击"退出"按钮
3. 返回登录页面

## 🗄️ 数据库结构

### users 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 用户 ID（主键） |
| username | VARCHAR(50) | 用户名（唯一） |
| password | VARCHAR(255) | 加密后的密码（salt$hash） |
| email | VARCHAR(100) | 邮箱（可选） |
| role | ENUM | 角色（teacher/student/admin） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 索引

- `idx_username`：用户名索引，加速登录查询

## 💡 常见问题

### Q1: 忘记密码怎么办？

目前版本暂不支持密码找回功能，建议：
- 记住密码或使用密码管理器
- 联系管理员重置密码

### Q2: 游客模式有什么限制？

游客模式**只能使用智能答疑功能**，其他功能均不可用：

**可用功能**：
- ✅ 智能答疑（提问和查看AI回复）

**不可用功能**：
- ❌ 课件生成（无法生成和下载课件）
- ❌ 学情分析（无法查看学情报告）
- ❌ 知识库管理（无法上传和管理文档）

**其他限制**：
- 无法保存聊天记录
- 无法导出学习数据
- 无法使用个性化功能
- 侧边栏会显示“游客模式”提示

### Q3: 如何修改密码？

当前版本需要在代码中调用 `auth_service.update_password()` 方法，后续版本将添加界面修改功能。

### Q4: 数据库中的密码是明文吗？

不是！密码使用 SHA-256 + Salt 加密存储，即使数据库泄露也无法直接获取明文密码。

### Q5: 可以批量导入用户吗？

可以！通过 SQL 语句批量插入，但需要注意密码必须使用 `auth_service.hash_password()` 加密。

## 🔧 开发接口

### AuthService 类

主要方法：

```python
from services.auth_service import auth_service

# 注册
result = auth_service.register_user(username, password, email, role)

# 登录
result = auth_service.login_user(username, password)

# 修改密码
result = auth_service.update_password(user_id, old_password, new_password)

# 获取用户信息
user = auth_service.get_user_by_id(user_id)

# 获取所有用户
users = auth_service.get_all_users()

# 删除用户
result = auth_service.delete_user(user_id)
```

### Session State

登录状态存储在 Streamlit session state 中：

```python
# 登录状态
st.session_state.logged_in = True/False

# 当前用户信息
st.session_state.current_user = {
    'id': 1,
    'username': 'test_teacher',
    'email': 'test@example.com',
    'role': 'teacher'
}
```
