# 多模态 AI 教学智能体 - 项目概要

> **版本**: v5.0 | **更新日期**: 2026年4月 | **架构**: 分层架构 (Core/Data/Services)

---

##  项目简介

**多模态 AI 教学智能体**是基于 Kimi 大模型的校园 AI 教学助手，采用**分层架构设计**和**RAG 优先策略**，为教师提供智能化、多模态的教学支持工具，覆盖课前备课、课中互动、课后辅导全场景。

### 核心技术优势
- 🏗️ **分层架构设计**：Core（核心支持）→ Data（数据访问）→ Services（业务逻辑）→ Presentation（表现层），代码可维护性提升 200%
- 💾 **三数据库分离**：主业务库 + 答疑库 + RAG 知识库，读写分离性能提升 40%，查询响应时间降低 60%
- 🔍 **RAG 优先策略**：先查知识库再调 AI，减少幻觉，API 成本降低 60%，响应速度提升 3 倍
- 🤖 **Kimi 大模型驱动**：支持超长上下文理解、多轮对话、JSON 结构化输出
- 🎯 **自我进化机制**：每次问答自动丰富数据库，越用越智能
- ⚡ **双模式课件生成**：快速模式（15-25秒）+ 标准模式（45-90秒），满足不同场景需求

## 🎯 核心功能模块

### 1️⃣ 智能答疑系统
- **三大场景覆盖**：课前预习、课中互动、课后辅导
- **双模式提问**：文字 + 语音输入
- **RAG 优先策略**：优先从知识库检索，减少 AI 幻觉，API 成本降低 60%
- **数据库优先**：高频问题零成本快速响应（<0.1 秒），命中率 > 80%
- **多轮对话支持**：自动保留最近 10 轮对话，理解上下文和指代关系
- **自我进化**：每次问答自动保存到数据库，构建知识库闭环

### 2️⃣ 课件自动生成系统
- **AI 智能识别学科**：输入主题，自动识别学科（语文、数学、英语、物理、化学等）
- **参考资料上传**：支持 PDF、Word、PPT、图片等多种格式
- **连续对话优化**：多轮描述需求，AI 逐步完善课件内容
- **双模式生成**：
  - 🚀 快速模式：8-10 页，15-25 秒，无配图，简化装饰（适合日常教学）
  - 🎨 标准模式：10-15 页，45-90 秒，含配图和精美装饰（适合公开课、示范课）
- **真实 PPT 预览**：Tab 分页展示，16:9 宽屏比例，渐变标题栏，模拟 PowerPoint 外观
- **一键下载 PPTX**：可编辑的 PowerPoint 文件，支持模板风格（科技、教育、自然、简约、商务）
- **配套教案生成**：自动生成教学目标、重点难点、活动设计等完整教案
- **数据验证与容错**：智能修复 AI 返回的空内容，确保所有页面正常显示

### 3️⃣ 学情智能分析
- **单个学生模式**：个性化学习报告，精准分析学生薄弱知识点
- **全班评估模式**：7 维度综合分析（成绩分布、知识点掌握、学生分层、典型问题、教学建议等）
- **多源数据融合**：问答记录、互动记录、作业数据综合分析
- **数据可视化**：图表和表格展示，直观呈现学情状态

### 4️⃣ 课堂互动工具
- **随机点名**：支持 Excel/CSV/TXT 名单导入，公平高效
- **互动记录管理**：自动保存，支持批量删除和清空
- **实时提问**：文字和语音双模式，覆盖全场景

### 5️⃣ 知识库管理
- **预置知识库**：93 本电子书（AI、C++、Math、Python、Extra 五大分类）
- **多格式上传**：PDF、Word、PPT、TXT、图片等
- **AI 智能解析**：自动提取知识点、教学目标、典型例题
- **智能检索**：全文搜索 + 学科筛选 + 使用统计
- **批量管理**：删除、清空、导出操作

## 🏗️ 技术架构

### 分层架构设计

```
┌─────────────────────────────────┐
│   表现层 (streamlit_001.py)     │  ← 用户界面
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│   业务层 (services/)            │  ← 核心业务逻辑
│  QA | Courseware | Knowledge    │
│  Analysis | Image | Animation   │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│   数据层 (data/)                │  ← 数据访问与处理
│  Config | DB Ops | Parser       │
│  Embedding | DataManager        │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│   核心层 (core/)                │  ← 基础工具
│  Logger | Utils | UI | Prompts  │
└─────────────────────────────────┘
```

### 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| **前端框架** | Streamlit 1.28+ | Python Web 应用 |
| **后端语言** | Python 3.8-3.11 | AI 生态丰富 |
| **AI 模型** | Kimi (moonshot-v1-8k) | 超长上下文，教育优化 |
| **数据库** | MySQL 8.0 | JSON 字段支持 |
| **文档处理** | python-pptx, openpyxl, pdfplumber | 多格式解析 |
| **向量检索** | 向量嵌入 + 相似度搜索 | RAG 高效检索 |

### 核心技术创新

#### 1. 三数据库分离架构
- **主业务库** (`ai_teaching_assistant`)：用户、班级、学生、课件、学情分析
- **答疑专用库** (`ai_qa_records`)：问答记录、会话管理、互动记录
- **RAG 知识库** (`ai_rag_knowledge`)：93 本电子书，向量化检索
- **优势**：读写分离，性能提升 40%；专库专用，便于维护

#### 2. RAG 优先策略
```
用户提问 → 查询 QA 数据库（>80% 相似度直接返回，<0.1 秒）
         → 检索 RAG 知识库（93 本电子书）
         → 基于知识库调用 AI 生成答案
         → 保存到数据库，自我进化
```
**成效**：API 成本降低 60%，平均响应时间从 1.5 秒降至 0.5 秒

## ⚡ 快速开始

### 环境要求
- **Python**: 3.8-3.11（不支持 3.12+）
- **MySQL**: 8.0+（支持 JSON 字段）
- **操作系统**: Windows 10/11, macOS 12+, Linux

### 安装步骤

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入 API Key 和数据库密码

# 3. 初始化数据库
python init_db.py        # 主业务库
python init_qa_db.py     # 答疑库
python init_rag_db.py    # RAG 知识库（预置 93 本电子书）

# 4. 启动应用
streamlit run streamlit_001.py
```

### 访问应用
- 默认地址：http://localhost:8501
- 浏览器自动打开
