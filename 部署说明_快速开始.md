# 🚀 快速部署指南

## 📋 系统要求

### 硬件要求
| 组件 | 最低配置 | 推荐配置 | 说明 |
|------|----------|----------|------|
| **CPU** | 双核 2.0 GHz | 四核 3.0 GHz+ | 无需 GPU |
| **内存** | 4 GB | 8 GB+ | 运行 Python + MySQL |
| **硬盘** | 10 GB 可用空间 | 50 GB+ | 数据库 + 日志 + 缓存 |
| **网络** | 稳定互联网连接 | 10 Mbps+ | 访问 Kimi API |

### 软件要求
| 软件 | 版本要求 | 下载地址 | 说明 |
|------|----------|----------|------|
| **Python** | 3.8 ~ 3.11 | https://www.python.org/ | ⚠️ 不支持 3.12+ |
| **MySQL** | 8.0+ | https://dev.mysql.com/downloads/ | 需要 JSON 字段支持 |
| **pip** | 20.0+ | Python 自带 | Python 包管理器 |

### 支持的操作系统
- ✅ **Windows**：Windows 10/11（64 位）
- ✅ **macOS**：macOS 12+ (Monterey)
- ✅ **Linux**：Ubuntu 20.04+ / CentOS 8+

### Python 版本说明
- ✅ **推荐版本**：Python 3.10.6（经过完整测试）
- ✅ **兼容版本**：Python 3.8 ~ 3.11
- ❌ **不支持版本**：Python 3.12+（部分依赖库不兼容）
- ❌ **不支持版本**：Python 3.7-（已停止维护）

**检查 Python 版本：**
```bash
python --version
# 应显示：Python 3.8.x ~ 3.11.x
```

### MySQL 版本说明
- ✅ **推荐版本**：MySQL 8.0.33+（支持 JSON 字段优化）
- ✅ **兼容版本**：MySQL 5.7+（部分功能受限）
- ❌ **不支持版本**：MySQL 5.6-（不支持 JSON 字段）

**检查 MySQL 版本：**
```bash
mysql --version
# 应显示：mysql  Ver 8.0.x 或 mysql  Ver 5.7.x
```

## 📦 安装步骤

### 1. 安装 Python 依赖

**前提条件**：
- ✅ Python 3.8 ~ 3.11 已安装
- ✅ pip 已安装（通常随 Python 一起安装）
- ✅ 网络连接正常（需要下载依赖包）

**安装依赖：**

```bash
# 方法一：使用 requirements.txt（推荐）
pip install -r requirements.txt

# 方法二：使用国内镜像加速（推荐国内用户）
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 方法三：手动安装核心依赖
pip install streamlit>=1.28.0 openai pandas openpyxl python-pptx mysql-connector-python python-dotenv
```

**核心依赖清单：**

| 依赖包 | 版本要求 | 用途 |
|--------|----------|------|
| **streamlit** | >=1.28.0 | Web 应用框架 |
| **openai** | Latest | Kimi API 兼容接口 |
| **pandas** | Latest | 数据处理与分析 |
| **openpyxl** | Latest | Excel 文件读写 |
| **python-pptx** | Latest | PPT 课件生成 |
| **mysql-connector-python** | Latest | MySQL 数据库驱动 |
| **python-dotenv** | Latest | 环境变量管理 |
| **pdfplumber** | Latest | PDF 文本提取（可选） |

**验证依赖安装：**
```bash
# 检查是否成功安装
pip list | grep streamlit
pip list | grep mysql-connector
```

---

### 2. 配置环境变量

#### 方法一：使用 .env 文件（推荐）

1. 复制环境变量模板：
```bash
copy .env.example .env
```

2. 编辑 `.env` 文件，填入实际配置：
```bash
# Kimi AI 配置
KIMI_API_KEY=your_actual_api_key_here
KIMI_BASE_URL=https://api.moonshot.cn/v1

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_actual_password
DB_NAME=ai_teaching_assistant

# 答疑数据库
QA_DB_HOST=localhost
QA_DB_PORT=3306
QA_DB_USER=root
QA_DB_PASSWORD=your_actual_password
QA_DB_NAME=ai_qa_records

# RAG 知识库（已预置 93 本电子书，覆盖 AI/C++/Math/Python/Extra 五大类）
RAG_DB_HOST=localhost
RAG_DB_PORT=3306
RAG_DB_USER=root
RAG_DB_PASSWORD=your_actual_password
RAG_DB_NAME=ai_rag_knowledge
```

#### 方法二：使用系统环境变量

**Windows (PowerShell):**
```powershell
$env:KIMI_API_KEY="your_api_key"
$env:DB_PASSWORD="your_password"
# 永久设置需要添加到系统环境变量
```

**Linux/Mac:**
```bash
export KIMI_API_KEY="your_api_key"
export DB_PASSWORD="your_password"
```

### 3. 初始化数据库

**前提条件**：
- ✅ MySQL 服务已启动
- ✅ MySQL root 用户密码已设置
- ✅ MySQL 版本 >= 5.7（推荐 8.0+）

**创建数据库（如未创建）：**
```bash
# 登录 MySQL
mysql -u root -p

# 创建三个数据库
CREATE DATABASE IF NOT EXISTS ai_teaching_assistant CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS ai_qa_records CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS ai_rag_knowledge CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 退出 MySQL
exit;
```

**运行初始化脚本：**
```bash
# 初始化主业务数据库（users, classes, students, courseware, learning_analysis）
python init_db.py

# 初始化答疑专用数据库（qa_records, qa_sessions, interaction_records）
python init_qa_db.py

# 初始化 RAG 知识库数据库（knowledge_documents）
python init_rag_db.py
```

**💡 提示**：
- RAG 数据库已预置 93 本电子书（AI, C++, Extra, Math, Python 五大类）
- 系统采用 RAG 优先策略：提问时先查知识库，减少 AI 调用
- 如需重新导入书籍，请运行 `python fix_import.py`（需安装 pdfplumber）
- 数据库字符集统一使用 `utf8mb4`（支持 emoji 和生僻字）

**验证数据库初始化：**
```bash
# 登录 MySQL 检查表结构
mysql -u root -p ai_teaching_assistant -e "SHOW TABLES;"
mysql -u root -p ai_qa_records -e "SHOW TABLES;"
mysql -u root -p ai_rag_knowledge -e "SHOW TABLES;"
```

### 4. 启动应用

**启动命令：**
```bash
# 进入项目目录
# 启动 Streamlit 应用
streamlit run streamlit_001.py
```



**常用启动参数：**
```bash
# 指定端口
streamlit run streamlit_001.py --server.port 8502

# 允许外部访问

streamlit run streamlit_001.py --server.headless true
```

**停止应用：**

## ⚠️ 安全建议

### 1. 不要提交敏感信息

`.gitignore` 已配置忽略以下文件：
- `.env` - 环境变量（包含密码和 API Key）
- `db_config.py` - 数据库配置
- `qa_db_config.py` - 答疑数据库配置
- `rag_db_config.py` - RAG 数据库配置

### 2. 使用强密码

```bash
# 生成随机密码示例
# Windows PowerShell
-join ((65..90) + (97..122) + (48..57) | Get-Random -Count 16 | ForEach-Object {[char]$_})

# Linux/Mac
openssl rand -base64 16
```

### 3. 限制数据库权限

```sql
-- 创建专用用户（不用 root）
CREATE USER 'ai_teaching'@'localhost' IDENTIFIED BY 'strong_password';

-- 只授予必要的权限
GRANT SELECT, INSERT, UPDATE, DELETE ON ai_teaching_assistant.* TO 'ai_teaching'@'localhost';
GRANT SELECT, INSERT, UPDATE, DELETE ON ai_qa_records.* TO 'ai_teaching'@'localhost';
GRANT SELECT, INSERT, UPDATE, DELETE ON ai_rag_knowledge.* TO 'ai_teaching'@'localhost';

FLUSH PRIVILEGES;
```

## 🔍 故障排查

### 数据库连接失败

1. 检查 MySQL 服务是否启动：
```bash
net start | findstr MySQL
```

2. 测试连接：
```bash
mysql -u root -p -h localhost -P 3306
```

3. 检查防火墙是否阻止 3306 端口

### API Key 无效

1. 确认 `.env` 文件存在且配置正确
2. 检查 API Key 是否复制完整
3. 确认 API Key 未过期

### 依赖安装失败

```bash
# 升级 pip
python -m pip install --upgrade pip

# 清除缓存重装
pip cache purge
pip install -r requirements.txt --force-reinstall
```
